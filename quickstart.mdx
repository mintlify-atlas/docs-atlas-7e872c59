## Overview

This guide will help you set up your development environment, build the smart contracts, and run the test suite for the Strat Perpetual DEX.

<Steps>
  <Step title="Install dependencies">
    First, ensure you have the Initia CLI (`initiad`) installed on your system.

    ```bash
    # Install initiad (follow Initia documentation for your platform)
    # Verify installation
    initiad version
    ```

    Clone the repository:

    ```bash
    git clone https://github.com/tansawit/contract.git
    cd contract
    ```
  </Step>

  <Step title="Configure named addresses">
    The contracts require named addresses to be specified during build and deployment. Update the addresses in the build command:

    ```bash
    initiad move build --named-addresses "strat=0x<YOUR_ADDRESS>,vip_score=0x<VIP_ADDRESS>,point=0x<POINT_ADDRESS>,stork=0x<STORK_ADDRESS>"
    ```

    <Tip>
    For development, you can use placeholder addresses. For production deployment, use your actual deployed contract addresses.
    </Tip>
  </Step>

  <Step title="Build the contracts">
    Build all Move modules:

    ```bash
    initiad move build --named-addresses "strat=0x3,vip_score=0x4,point=0x5,stork=0x6"
    ```

    This compiles all modules and checks for compilation errors.
  </Step>

  <Step title="Run tests">
    Execute the comprehensive test suite:

    ```bash
    initiad move test --dev
    ```

    This runs all tests including:
    - Order creation and execution
    - Position management
    - Vault operations
    - Oracle integration
    - Fee calculations
    - Governance functions

    <Note>
    Tests use dev-addresses defined in Move.toml for local testing.
    </Note>
  </Step>
</Steps>

## Common commands

### Build specific module

```bash
initiad move build --named-addresses "strat=0x3,vip_score=0x4,point=0x5,stork=0x6"
```

### Run specific test

```bash
initiad move test --dev -f test_order
```

### Run tests with verbose output

```bash
initiad move test --dev -v
```

### Verify dependencies

```bash
initiad move verify
```

## Troubleshooting

<Accordion title="Named address errors">
  If you see errors about undefined named addresses, ensure all required addresses are specified:
  - `strat`: Main protocol address
  - `vip_score`: VIP scoring system
  - `point`: Points/rewards system
  - `stork`: Stork oracle address
</Accordion>

<Accordion title="Test failures">
  If tests fail:
  1. Ensure you're using `--dev` flag for development addresses
  2. Check that dependencies (InitiaStdlib, Stork) are accessible
  3. Verify your Initia CLI version is up to date
</Accordion>

<Accordion title="Build errors">
  Common build issues:
  - Missing dependencies: Run `initiad move verify` to check
  - Named address mismatch: Verify all addresses are specified correctly
  - Syntax errors: Check the error output for specific file and line numbers
</Accordion>
