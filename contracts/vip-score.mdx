The VIP score module implements a comprehensive scoring system that rewards users based on their trading volume and vault deposits. Users accumulate points each stage, which determine their VIP tier and associated fee discounts.

## Overview

The VIP scoring system tracks user activity across time-bound stages and calculates scores based on:
- **Trading Volume**: Raw trading volume with asset-specific and collateral-specific multipliers
- **Vault Deposits**: LP token deposits with duration-based multipliers
- **Stage Management**: Automatic stage transitions and score finalization

## Key Concepts

### Stages

Time-bound periods during which scores are accumulated. Each stage:
- Has a start time, end time, and interval
- Accumulates separate scores for each user
- Is finalized when the stage ends
- Automatically transitions to the next stage

### Score Calculation

Scores are calculated using a complex formula that considers:

```
score = (trade_volume * trade_asset_multiplier * trade_collateral_multiplier * vault_multiplier) / trade_factor

vault_multiplier = min(max_vault_multiplier, 1 + vault_collateral_multiplier * deposit_amount * duration_multiplier / deposit_factor)

duration_multiplier = min(max_duration_multiplier, 1 + time_weighted_lp_balance / duration_factor)
```

## Data Structures

### ModuleStore

Main storage for stage and deployer management:
- `init_stage`: The initial stage number
- `deployers`: Authorized deployers who can feed events
- `scores`: Mapping of stage numbers to score data

### StratDataStore

Stores configuration and user data for VIP scoring:
- `paused`: Whether score updates are paused
- `decimal_scores`: User decimal scores (to minimize rounding)
- `lp_prices`: LP token prices for each collateral
- `vault_data`: Time-weighted vault deposit data
- `trade_config`: Trading score configuration
- `vault_configs`: Vault deposit configuration per collateral

### StageInfo

Stage timing information:
- `stage`: Current stage number
- `stage_start_time`: Start timestamp of current stage
- `stage_end_time`: End timestamp of current stage
- `stage_interval`: Duration of each stage in seconds

## Score Multipliers

### Vault Multiplier

Rewards users for depositing collateral in vaults:
- Increases with deposit amount
- Increases with deposit duration (time-weighted)
- Capped at `max_vault_multiplier`
- Resets to 1.0 on withdrawal

### Duration Multiplier

Rewards long-term deposits:
- Based on time-weighted LP balance
- Calculated as: `1 + time_weighted_balance / duration_factor`
- Capped at `max_duration_multiplier`

### Asset Multiplier

Different assets can have different score weights:
- Configured per asset ID
- Default multiplier applies to unlisted assets

### Collateral Multiplier

Different collaterals can have different score weights:
- Configured per collateral metadata
- Default multiplier applies to unlisted collaterals
