---
title: "Oracle Aggregator Module"
description: "Price feed aggregation from Stork oracle with market status tracking"
---

The `oracle_aggregator` module aggregates price feeds from the Stork oracle and manages market status for trading pairs, including market hours and status transitions.

## Core Data Structures

### OracleType

Enum representing supported oracle types.

```move
enum OracleType has copy, drop, store {
    Stable,
    Stork
}
```

<ResponseField name="Stable" type="variant">
  Stable price oracle (always returns 1.0)
</ResponseField>

<ResponseField name="Stork" type="variant">
  Stork oracle integration for real-time price feeds
</ResponseField>

### OracleKey

Identifies an oracle feed.

<ResponseField name="type" type="OracleType">
  Type of oracle (Stable or Stork)
</ResponseField>

<ResponseField name="id" type="vector<u8>">
  Oracle-specific identifier
</ResponseField>

### MarketStatus

Represents the current and upcoming market status.

<ResponseField name="next_change_time" type="u64">
  Timestamp when market status will change
</ResponseField>

<ResponseField name="current_status" type="u8">
  Current market status code
</ResponseField>

<ResponseField name="next_status" type="u8">
  Upcoming market status code after next_change_time
</ResponseField>

### MarketStatusKeys

Oracle feed identifiers for market status data.

<ResponseField name="type" type="OracleType">
  Oracle type providing market status
</ResponseField>

<ResponseField name="next_change_time" type="vector<u8>">
  Oracle ID for next change time feed
</ResponseField>

<ResponseField name="current_status" type="vector<u8>">
  Oracle ID for current status feed
</ResponseField>

<ResponseField name="next_status" type="vector<u8>">
  Oracle ID for next status feed
</ResponseField>

## Entry Functions

### update_market_status_stork

Updates market status from Stork oracle feeds.

```move
public entry fun update_market_status_stork(
    account: &signer,
    asset_id: u64,
    temporal_numeric_value_timestamp_ns: vector<u64>,
    temporal_numeric_value_magnitude: vector<u128>,
    temporal_numeric_value_negative: vector<bool>,
    publisher_merkle_roots: vector<vector<u8>>,
    value_compute_alg_hashes: vector<vector<u8>>,
    rs: vector<vector<u8>>,
    ss: vector<vector<u8>>,
    vs: vector<u8>
)
```

<ParamField path="account" type="&signer" required>
  Account submitting the oracle update
</ParamField>

<ParamField path="asset_id" type="u64" required>
  Strat asset ID (not Stork asset ID)
</ParamField>

<ParamField path="temporal_numeric_value_timestamp_ns" type="vector<u64>" required>
  Timestamps in nanoseconds for the three oracle values
</ParamField>

<ParamField path="temporal_numeric_value_magnitude" type="vector<u128>" required>
  Magnitudes for next_change_time, current_status, and next_status
</ParamField>

<ParamField path="temporal_numeric_value_negative" type="vector<bool>" required>
  Sign indicators for the oracle values
</ParamField>

<ParamField path="publisher_merkle_roots" type="vector<vector<u8>>" required>
  Merkle roots from Stork publishers
</ParamField>

<ParamField path="value_compute_alg_hashes" type="vector<vector<u8>>" required>
  Algorithm hashes for value computation
</ParamField>

<ParamField path="rs" type="vector<vector<u8>>" required>
  ECDSA signature r components
</ParamField>

<ParamField path="ss" type="vector<vector<u8>>" required>
  ECDSA signature s components
</ParamField>

<ParamField path="vs" type="vector<u8>" required>
  ECDSA signature v components
</ParamField>

<Note>
This function includes validation:
- Timestamps must match across all three values
- Oracle data must be recent (within 10 seconds)
- Cannot update during market status transition buffer period
</Note>

## Oracle Integration

### Stork Oracle

The module integrates with the Stork oracle for real-time price feeds. Stork prices are:

- Provided with nanosecond timestamps
- Scaled by 10^18 (converted to BigDecimal)
- Verified using ECDSA signatures from multiple publishers
- Subject to recency checks

### Stable Oracle

The stable oracle always returns:
- Price: `1.0`
- Timestamp: Current block timestamp

Useful for stablecoin collateral or testing.

## Market Status Transition Buffers

To prevent trading during volatile periods around market status changes, the system enforces transition buffers:

- **Pre-transition buffer**: Period before market status change when oracle updates are blocked
- **Post-transition buffer**: Period after market status change when oracle updates are blocked

Default: 60 seconds for both buffers

<Warning>
Operations requiring price feeds are not allowed when:
`market_change_timestamp - pre_buffer ≤ oracle_timestamp ≤ market_change_timestamp + post_buffer`
</Warning>
