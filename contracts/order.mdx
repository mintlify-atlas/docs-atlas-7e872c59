---
title: "Order Module"
description: "Core trading interface for creating and managing perpetual orders"
---

The `order` module provides the primary user interface for trading on the Strat Perpetual DEX. Users can create market, limit, and stop orders, manage positions, and handle collateral.

## Opening Positions

### create_market_open_order

Creates a market order to open a position, executed immediately.

```move
public entry fun create_market_open_order(
    account: &signer,
    asset_id: u64,
    collateral_metadata: Object<Metadata>,
    collateral_amount: u64,
    target_price: BigDecimal,
    tp_price: BigDecimal,
    sl_price: BigDecimal,
    leverage: BigDecimal,
    is_long_position: bool,
    max_slippage: BigDecimal
)
```

<ParamField path="account" type="&signer" required>
  User account signer
</ParamField>

<ParamField path="asset_id" type="u64" required>
  ID of the asset to trade
</ParamField>

<ParamField path="collateral_metadata" type="Object<Metadata>" required>
  Metadata object for the collateral token
</ParamField>

<ParamField path="collateral_amount" type="u64" required>
  Amount of collateral to deposit
</ParamField>

<ParamField path="target_price" type="BigDecimal" required>
  Expected execution price (for slippage checking)
</ParamField>

<ParamField path="tp_price" type="BigDecimal" required>
  Take profit price (0 for no TP)
</ParamField>

<ParamField path="sl_price" type="BigDecimal" required>
  Stop loss price (0 for no SL)
</ParamField>

<ParamField path="leverage" type="BigDecimal" required>
  Desired leverage multiplier
</ParamField>

<ParamField path="is_long_position" type="bool" required>
  True for long, false for short
</ParamField>

<ParamField path="max_slippage" type="BigDecimal" required>
  Maximum acceptable slippage
</ParamField>

## Events

The order module emits events for all state changes:

- `CreateOpenOrderEvent` - When an open order is created
- `UpdateOpenOrderEvent` - When an open order is updated
- `DeleteOpenOrderEvent` - When an open order is cancelled or executed
- `CreateCloseOrderEvent` - When a close order is created
- `DeleteCloseOrderEvent` - When a close order is cancelled or executed
- `OpenPositionEvent` - When a position is opened
- `ClosePositionEvent` - When a position is closed
- `UpdatePositionEvent` - When position TP/SL is updated
- `AddCollateralEvent` - When collateral is added to a position
- `CreateRemoveCollateralOrderEvent` - When a remove collateral order is created
- `DeleteRemoveCollateralOrderEvent` - When a remove collateral order is cancelled or executed
