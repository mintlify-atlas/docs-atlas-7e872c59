The signed_bigdecimal module provides a comprehensive library for working with signed decimal numbers. It wraps the standard BigDecimal type to add support for negative values and provides a full suite of arithmetic and comparison operations.

## Overview

This library is essential for financial calculations in the perpetual DEX where positions can have both positive and negative values (profits and losses, long and short positions, etc.).

## Data Structure

### SignedBigDecimal

```move
struct SignedBigDecimal has copy, drop, store {
    magnitude: BigDecimal,
    is_positive: bool
}
```

A signed decimal number represented by:
- `magnitude`: The absolute value as a BigDecimal
- `is_positive`: Sign flag (true for positive/zero, false for negative)

**Note**: Zero is always represented with `is_positive = true`

## Arithmetic Operations

### add

Add two signed decimals.

```move
public fun add(self: &SignedBigDecimal, other: &SignedBigDecimal): SignedBigDecimal
```

<ParamField path="self" type="&SignedBigDecimal">
  First operand
</ParamField>

<ParamField path="other" type="&SignedBigDecimal">
  Second operand
</ParamField>

**Logic**:
- Same sign: Add magnitudes, keep sign
- Different signs: Subtract smaller from larger, use sign of larger
- Equal magnitudes with different signs: Return zero

## Usage Examples

### Basic Arithmetic

```move
let a = from_u64(5);        // 5
let b = from_u64(3);        // 3
let c = a.add(&b);          // 8
let d = a.sub(&b);          // 2
let e = a.mul(&b);          // 15
let f = a.div(&b);          // 1.666...
```

### Working with Negative Numbers

```move
let positive = from_u64(10);           // 10
let negative = positive.negate();       // -10
let sum = positive.add(&negative);      // 0
let diff = positive.sub(&negative);     // 20
```

### Comparison

```move
let a = from_u64(5);
let b = from_u64(3).negate();  // -3

assert!(a.gt(&b), 0);   // 5 > -3
assert!(b.lt(&zero()), 0);     // -3 < 0
```

### Min/Max

```move
let a = from_u64(5);
let b = from_u64(3).negate();
let minimum = a.min(&b);  // -3
let maximum = a.max(&b);  // 5
```

## Error Codes

- `ENOT_POSITIVE (1)`: Attempted to convert negative value to unsigned BigDecimal

## Implementation Notes

1. **Zero Normalization**: Zero is always stored with `is_positive = true` to ensure consistency
2. **Precision**: Inherits precision from the underlying BigDecimal type
3. **No Overflow**: Uses BigDecimal internally, so no overflow concerns for typical use cases
4. **Immutable**: All operations return new values rather than modifying in place
