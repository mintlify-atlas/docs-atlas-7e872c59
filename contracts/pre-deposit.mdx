The pre-deposit points module implements a reward system for early depositors who lock or deposit collateral before the main protocol launch. Users earn points based on their deposit amounts and duration.

## Overview

This module provides two distinct point earning mechanisms:
- **Deposit Points**: Earned by depositing collateral to the module
- **Lock Points**: Earned by locking collateral in the vault system

Both systems use time-based emission with daily emission rates and accumulate points proportionally to deposit amounts and duration.

## Key Features

- **Dual Point Systems**: Separate tracking for deposits and locks
- **Time-based Rewards**: Points accrue over time based on daily emission rates
- **Amount Caps**: Configurable maximum deposit amounts
- **Close Mechanism**: Admin can set a close timestamp to freeze points
- **Auto-withdrawal**: Admin can batch withdraw deposits for users after close

## Data Structures

### PointStore

Main storage structure:
- `collateral_metadata`: The collateral token being tracked
- `close_timestamp`: Timestamp when deposits are frozen (0 if not set)
- `deposit_point`: Point data for direct deposits
- `lock_point`: Point data for vault locks

### Point

Point accumulation data:
- `user_points`: Mapping of users to their point data
- `global_index`: Global accumulator for point distribution
- `last_updated_timestamp`: Last time global index was updated
- `daily_emission`: Points emitted per day
- `total_amount`: Total amount deposited across all users
- `amount_cap`: Maximum total amount allowed

### UserPoint

Per-user point data:
- `point`: Accumulated points (BigDecimal for precision)
- `amount`: Current deposit/lock amount
- `last_index`: User's checkpoint of global index

## Point Calculation

Points are calculated using a global index system:

### Global Index

The global index accumulates over time:

```
global_index += (daily_emission * time_diff) / (DAY * total_amount)
```

Where:
- `daily_emission`: Points emitted per day
- `time_diff`: Time elapsed since last update (seconds)
- `DAY`: 86400 seconds
- `total_amount`: Total deposits across all users

### User Points

When a user's points are calculated:

```
user_point += (global_index - user_last_index) * user_amount
```

This ensures:
- Points are proportional to deposit amount
- Points accrue over time
- Fair distribution based on share of total deposits

## Close Mechanism

The close mechanism allows the admin to freeze deposits:

1. **Before Close**: Users can deposit, lock, and withdraw freely
2. **Set Close**: Admin sets a future close timestamp
3. **After Close**: 
   - No deposits, locks, or withdrawals allowed
   - Points continue to accrue until close timestamp
   - Admin can batch withdraw for users

**Point Calculation After Close**:
- If current time > close_timestamp, use close_timestamp for calculations
- This freezes point accumulation at the close time

## Constants

- `DAY`: 86400 seconds (60 * 60 * 24)
