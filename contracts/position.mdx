---
title: "Position Module"
description: "Position tracking, PnL calculations, and position lifecycle management"
---

The `position` module manages all open positions in the Strat Perpetual DEX. It handles position creation, updates, PnL calculations, and maintains the relationship between collateral, leverage, and trade size.

## Friend Functions

These functions are called by the `order` module during position lifecycle operations.

### add_position

Creates a new position or merges with an existing position.

```move
public(friend) fun add_position(
    user_addr: address,
    asset_id: u64,
    collateral_metadata: Object<Metadata>,
    collateral_amount: u64,
    open_price: BigDecimal,
    tp_price: BigDecimal,

    acc_rollover_fee: SignedBigDecimal,
    acc_funding_fee: SignedBigDecimal,
    size: BigDecimal,
    is_long_position: bool
)
```

When merging positions:
- Open price is size-weighted average
- Leverage is collateral-weighted average
- TP/SL are updated to new values
- Fees are recalculated

<ResponseField name="size_closed" type="BigDecimal">

### add_collateral

Adds collateral to an existing position, reducing leverage.

```move
public(friend) fun add_collateral(
    user_addr: address,
    asset_id: u64,
    collateral_metadata: Object<Metadata>,
    collateral_amount: u64,
    is_long_position: bool
)
```

Leverage calculation:
```
new_leverage = (old_leverage * old_collateral) / (old_collateral + added_collateral)
```

### remove_collateral

Removes collateral from a position, increasing leverage.

```move
public(friend) fun remove_collateral(
    user_addr: address,
    asset_id: u64,
    collateral_metadata: Object<Metadata>,
    collateral_remove_amount: u64,
    is_long_position: bool
)
```

Leverage calculation:
```
new_leverage = (leverage * old_collateral) / (old_collateral - removed_collateral)
```

### update_tp_and_sl_price

Updates take profit and stop loss prices for a position.

```move
public(friend) fun update_tp_and_sl_price(
    user_addr: address,
    asset_id: u64,
    collateral_metadata: Object<Metadata>,

    new_sl_price: BigDecimal
```

Validation rules:
- For long positions: TP must be greater than SL (if both set)
- For short positions: TP must be less than SL (if both set)

## PnL Calculations

### current_pnl_by_price_changed

Calculates unrealized PnL based on current price, capped at max_pnl_ratio.

```move
public fun current_pnl_by_price_changed(
    self: &Position,
    price: BigDecimal
): (SignedBigDecimal, bool)
```

<ParamField path="self" type="&Position" required>
  Position reference
</ParamField>

<ParamField path="price" type="BigDecimal" required>
  Current market price
</ParamField>

<ResponseField name="pnl" type="SignedBigDecimal">
  Unrealized PnL (capped at max if applicable)
</ResponseField>

<ResponseField name="is_max_pnl" type="bool">
  True if PnL was capped at maximum
</ResponseField>

PnL calculation:
```
For long:  pnl = (price - open_price) * size
For short: pnl = (open_price - price) * size

max_pnl = (open_price * max_pnl_ratio / leverage) * size
```

## Position Merging

When a user opens a new position in the same direction for the same asset-collateral pair:

1. **Open Price**: Size-weighted average
   ```
   new_open_price = (old_open * old_size + new_open * new_size) / (old_size + new_size)
   ```

2. **Leverage**: Collateral-weighted average
   ```
   new_leverage = (old_lev * old_coll + new_lev * new_coll) / (old_coll + new_coll)
   ```

3. **Fees**: Accumulated and recalculated

4. **TP/SL**: Updated to new values and auto-corrected if needed

## Events

The position module emits `PositionChangeEvent` for all state changes:

```move
#[event]
struct PositionChangeEvent has copy, drop {
    event_type: String,  // "create", "update", or "delete"
    position_id: u64,
    position_key: PositionKey,
    position: Option<PositionResponse>  // None for delete events
}
```

## Take Profit Auto-Correction

To prevent excessive profits, TP prices are automatically capped:

```
max_tp_diff = open_price * max_pnl_ratio / leverage

For long:  max_tp = open_price + max_tp_diff
For short: max_tp = max(0, open_price - max_tp_diff)
```

This ensures PnL cannot exceed `max_pnl_ratio` times the initial margin.
