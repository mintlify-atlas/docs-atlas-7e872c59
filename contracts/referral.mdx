The referral module implements a comprehensive referral code system that allows traders to generate unique codes and earn rewards by inviting new users to the platform.

## Overview

The referral system uses a base36-encoded 8-character code format (e.g., "A7NUHBI8") generated through double hashing to ensure uniform distribution and collision avoidance. Users must meet a minimum trading volume requirement before they can generate their own referral code.

## Key Features

- **Volume-based Code Generation**: Users must reach a minimum trading volume before generating referral codes
- **Signature Verification**: EIP-191 and SECP256k1 signature verification for secure code registration
- **Collision-resistant Hashing**: Double hashing (SHA3-256 and SHA2-256) with linear probing
- **User Volume Tracking**: Automatic accumulation of trading volume for all users

## Data Structures

### ModuleStore

The main storage structure containing:
- `volume_requirement`: Minimum volume needed to generate a referral code
- `user_volumes`: Mapping of user addresses to their total trading volumes
- `referral_codes`: Mapping of code (u64) to referee addresses
- `referee_to_code`: Mapping of referee addresses to their codes
- `referral_map`: Mapping of user addresses to their referrers

## Code Generation Algorithm

The referral code generation uses a sophisticated algorithm:

1. **Hashing**: Compute SHA3-256 and SHA2-256 of the user's address
2. **Modular Arithmetic**: Apply modulo 2^41 to both hash values
3. **Probing**: Use linear probing with odd step size to find unused codes
4. **Encoding**: Convert the final u64 value to base36 (8 characters, A-Z0-9)

This ensures:
- ~2 trillion possible codes (between 2^41 and 2^42)
- Uniform distribution across the code space
- Minimal collision probability

## Security

The module implements multiple security measures:

- **EIP-191 Signature Verification**: Ensures the user owns the address
- **SECP256k1 Recovery**: Derives address from public key using Keccak256
- **Code Validation**: Verifies referral codes exist before registration
- **Governance Controls**: Volume requirements can only be updated by governance
